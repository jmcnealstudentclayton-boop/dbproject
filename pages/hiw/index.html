Reviews · Project Overview

How the Review Systems Work

This portfolio walks through the same review data model implemented three different ways: Google Sheets, a local MySQL + PHP stack, and Supabase (hosted Postgres).

Each page below explains the tools, schema, and data flow — from user form submission all the way to live review summaries.

Same Data · Different Stacks

  • • Shared concept of items, customers, and reviews.
  • • Focus on normalized tables and summary views.
  • • Shows how SQL skills translate between tools and hosting environments.

Use this page as a map: pick the stack you’re interested in, then jump into its detailed “How It Works” page.

Option 1

Google Sheets + Make + JSONBin

A “no-code / low-code” database built with Google Sheets as the backend, Make (Integromat) for automation, and JSONBin to feed reviews to the website.

  • • Web form → Make webhook → Google Sheets
  • • Array formulas generate demo IDs (e.g. cus_001).
  • • Make converts rows to JSON and publishes to JSONBin.
Great for: quick prototypes View Details

Option 2

Local SQL (MySQL + PHP + XAMPP)

A classic server-side stack using MySQL, dBeaver for database management, and PHP running on XAMPP to handle form submissions.

  • • XAMPP runs Apache + MySQL locally.
  • • dBeaver manages schemas, tables, and views.
  • • PHP receives POST requests and runs SQL INSERT queries.
Great for: backend fundamentals View Details

Option 3

Supabase (Hosted Postgres)

A modern cloud option where Supabase hosts a full Postgres database and exposes it through secure REST and JavaScript APIs.

  • • Tables + views created in Supabase SQL editor.
  • • Front-end calls supabase.from("reviews") directly.
  • • Same schema and views as the MySQL project, just in the cloud.
Great for: production-style hosting View Details

Quick Comparison

Hosting

Google Sheets lives in the cloud, but works like a spreadsheet backend.

MySQL + PHP runs on your own machine via XAMPP.

How Reviews Are Stored

Rows in Google Sheets, generated IDs, summary sheets for stats.

Relational tables and views in MySQL, managed in dBeaver.

Relational tables and views in hosted Postgres via Supabase.
