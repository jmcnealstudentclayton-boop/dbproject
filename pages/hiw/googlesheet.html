Step 0

Planning the Data Fields

Before anything was built, we first listed what kind of information we wanted to collect and connect.

  • • Review ID, Product ID, Customer Info
  • • Ratings & Review Text
  • • Item Details (Item Name, Cost)
  • • Dates & Summary Stats

This helped define what the columns and tables would be later in Google Sheets.

Step 1

Normalizing Data in Google Sheets

We then split the data into several related sheets (tables) to follow database normalization rules.

  • • Reviews — where form submissions go
  • • Customers — stores user info and creation date
  • • Items — holds product details
  • • Customer Review Summary — counts total reviews per customer
  • • Item Review Summary — average score, total reviews, first/last review date

Each sheet is connected through shared IDs like customer_id and item_id.

Step 2

Building the ERD (Entity Relationship Diagram)

With the tables ready, we visualized their relationships using Draw.io.

Each entity (sheet) connects via primary and foreign keys, ensuring structured, linked data — similar to a SQL relational database.

  • • customer_id connects reviews ↔ customers
  • • item_id connects reviews ↔ items

Step 3

User Submits the Review Form

On the website, a customer fills out:

  • • First & Last Name
  • • Product / Item ID
  • • Star Rating
  • • Review Text

When they hit Submit, the form sends all fields to a Make webhook URL.

Step 4

Make Receives the Webhook & Adds Metadata

The Make scenario starts with a Custom Webhook module that receives the payload from the website.

Tools · Variables

Generate extra fields

  • rev_id — random review ID like rev_2048
  • submit_date — date of submission

Google Sheets · Add a Row

Combine form + variables

Make sends all values (form fields + rev_id + submit_date) into the next empty row in the Reviews sheet.

Step 5

Google Sheets Stores the Reviews

Each submission becomes a new row with columns like:

  • • review_id (from Make)
  • • customer_id (generated in-sheet)
  • • item_id, first_name, last_name
  • • star_score, review_text, review_date

Customer ID Formula (Demo Only) 

=ARRAYFORMULA(
  IF(A2:A<>"",
     "cus_" & TEXT(
       MATCH(ROW(A2:A), FILTER(ROW(A2:A), A2:A<>""), 0),
     "000"),
  "")
)

This generates demo IDs like cus_001, cus_002, etc. It’s not meant to be perfect — just simple for this project.

Step 6

Make Converts Rows to JSON & Sends to JSONBin

Another part of the Make scenario:

  • • Get Range Values from the review sheet
  • • Use an Array Aggregator to build an array of review objects
  • • Use the JSON module to create the payload
  • • Send it via HTTP > Make a request to JSONBin

JSONBin Endpoint

Step 7

Website Reads JSON & Renders Live Summaries

The front-end fetches the JSON from JSONBin and builds two visual summaries:

  • • Per Product — average rating, review count, recent review snippets
  • • Per Customer — number of reviews, average rating, products reviewed

Because everything runs off the same JSON file, the visuals update automatically whenever new data is pushed from Google Sheets via Make.
