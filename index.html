Choose a path

Same reviews, different backends.

Each path handles the same “submit a review” flow, but the tech and trade-offs are different. Use this as a side-by-side comparison when presenting.

Google Sheets + Make

No server, all automation.

Web form → Make webhook → Google Sheets → JSONBin. The live site reads JSON and draws real-time summaries.

  • • Easiest to deploy (GitHub Pages friendly)
  • • Great for demos & quick dashboards
  • • Limited by sheet size & API calls

Supabase

Hosted Postgres with batteries included.

Real database in the cloud with auth, row-level security, and SQL. This project mirrors the same review schema.

  • • Strong choice for production apps
  • • SQL + REST + realtime subscriptions
  • • Needs key management & backend rules

SQL + PHP (XAMPP)

Classic LAMP-style stack, local only.

MySQL schema, views, and a small PHP script running on XAMPP. Perfect for learning how web servers talk to databases.

  • • Full control over SQL & constraints
  • • Requires local server, not GitHub-friendly
  • • Great for explaining backend flow

At a glance

A lightweight summary of when you might reach for each approach in a real project.

Stack Best for Pros Trade-offs
Google Sheets + Make Low-code demos, quick prototypes No server, easy to visualize, great for non-devs Scaling & structure limited to Sheets
Supabase Serious apps with auth & APIs Managed Postgres, rich features, modern tooling More config (keys, security rules, pricing)
SQL + PHP (XAMPP) Learning full server flow, local experiments Total control, classic stack, great teaching tool Not deployable on GitHub Pages; more setup
